# Single Session Report Template

Use this structure for a **formal Chinese single-session report**. The goal is not to sound academic. The goal is to produce a psychologically serious review that is readable now and still useful when the user comes back months later.

## Core stance

- write in Chinese
- be clinically thoughtful, not inflated
- reconstruct the process, not just the topic list
- separate facts from interpretations
- mark uncertainty explicitly
- do not diagnose without evidence
- prefer a few strong working hypotheses over one totalizing story

## Before writing

Make sure you have:

- the session page
- the best available transcript source
  - prefer `transcript.edited.md` when present
  - otherwise use `transcript.turns.md`
- the review file if transcript quality or speaker attribution is shaky
- relevant prior session pages when continuity matters

## Recommended structure

### 1. 本次会谈最核心的工作点

用 3-6 句话回答：

- 这次会谈真正围绕什么工作点展开
- 为什么这是本次最重要的核心
- 相比前几次，这次最值得注意的新东西是什么

这一段应该像“临床摘要”，不是主题标签列表。

### 2. 会谈过程脉络

按过程写，不按主题拼盘写。建议至少覆盖：

- 开场状态：来访者是带着怎样的情绪组织、关系姿态或叙事进入会谈的
- 激活点：哪一个片段把情绪、关系或核心冲突真正带出来了
- 中段转折：哪里开始靠近核心，哪里又撤离、理智化或收缩
- 结尾落点：会谈最后停在了哪里，是更清楚了，还是更悬而未决

重点是让读者看见“会谈是怎么走到这里的”。

### 3. 来访者状态与心理线索

这一部分建议按以下维度组织，但不要机械套框：

- 主导情绪与情绪转折
- 防御与回避方式
- 自我叙事与自我评价
- 关系姿态与被看见 / 被理解 / 被拒绝的期待或害怕
- 羞耻、控制、安全感、依赖、愤怒、无力、讨好、顺从等关键维度

这里必须区分：

- 可以直接观察到的事实
- 基于这些事实提出的理解

### 4. 咨询师干预与其可能作用

写清楚：

- 咨询师实际做了什么
- 哪些回应帮助来访者更接近体验
- 哪些回应可能过快、过浅、过认知化，或让过程变平
- 哪些地方值得在下一次会谈中调整节奏或跟进方式

如果没有足够证据，不要把咨询师意图写成事实。

### 5. 与既往咨询的连续性或变化

回答两个问题：

- 这次重复了哪些熟悉的主题、姿态或关系模式
- 这次出现了哪些新的移动、松动、命名或反复

要让人看见这不是一份“孤立 session 摘要”，而是放在工作过程里的定位。

### 6. 当前工作假设

写 2-4 条，不要只写一条大总论。每条假设都要说明：

- 支持它的证据
- 可能的替代解释
- 为什么它目前仍只是工作假设

优先写那些最能指导后续工作的假设。

### 7. 下次可继续探索的问题

尽量写成可以带进下一次会谈的问题或切入口，而不是空泛建议。优先包括：

- 哪个情绪或关系现场值得更多停留
- 哪个防御最值得轻轻碰一下
- 哪个叙事需要从“知道”走向“真正接触到”

### 8. 不确定性说明

明确写出：

- 哪些判断仍依赖更多材料
- 哪些地方 transcript / speaker attribution 可能影响理解
- 哪些地方容易被过度解释

### 9. 证据附录

列出最关键的引用片段：

- `session_id + timestamp`
- speaker
- 原句或简短摘录
- 它支持的是哪一条观察或假设

## Report quality rubric

一份合格的单次复盘，应该满足下面这些标准：

- 读起来像一个有经验的人在复盘会谈，而不是在堆心理学术语
- 能让人看见“过程脉络”，而不是只看见主题标签
- 至少出现一次明确的不确定性说明
- 至少提出 2 条真正可工作的假设
- 至少引用 4-8 个关键片段
- 不把支持、解释、督导和朋友式提醒混成一种声音
